RESTful API 軟體架構範本

基於分層架構 (Layered Architecture) + Port/Adapter 模式

WEB Web Layer（表現層）

負責接收 HTTP 請求、驗證輸入、轉換為內部物件，並呼叫 Application Layer 處理業務。回傳 Response DTO 給客戶端。

web/
  XxxController.java ── REST 端點，每個 Use Case 一個 Controller
  port/
    XxxRequest.java ── 請求 DTO（含 Validation）
    XxxResponse.java ── 回應 DTO
▼ 呼叫
APPLICATION Application Layer（應用層）

協調業務流程，調度 Domain 物件和 Repository。透過 Port (Interface) 定義對外依賴，不直接依賴具體實作。

application/
  XxxService.java ── 業務流程協調者
  port/ ── 定義 Repository 的抽象介面
    XxxPort.java ── Interface，由 Repository 實作
  dto/ ── 層間傳遞用物件
    XxxDto.java ── 資料傳輸物件 / Value Object
▼ 使用 Domain 物件 ＆ 呼叫 Port
DOMAIN Domain Layer（領域層）

核心業務邏輯。包含 Entity、Value Object、Aggregate Root。不依賴任何框架或基礎設施，純粹表達業務規則與行為。

domain/
  XxxEntity.java ── 實體，含業務行為與狀態
  XxxValueObject.java ── 值物件（不可變）
▲ 被 Application 透過 Port 呼叫
REPOSITORY Repository Layer（基礎設施層）

實作 Application Port 定義的介面。負責實際的資料存取（DB、Redis、外部 API 等），將技術細節隔離在此層。

repository/
  XxxRepo.java ── 實作 application/port 中的 Interface

🔗 層間依賴規則

  • Web → Application：Controller 注入 Service，將 Request DTO 轉為 Domain 物件後呼叫 Service 方法
  • Application → Domain：Service 使用 Domain Entity/Value Object 來表達業務邏輯
  • Application → Port (Interface)：Service 透過注入的 Port 介面操作資料，不知道具體實作
  • Repository → Application Port：Repository 類別 implements Port 介面，提供具體資料存取實作
  • 禁止反向依賴：Domain 不依賴任何其他層；Application 不依賴 Web 或 Repository 具體類別

🔄 請求處理流程

HTTP Request
客戶端發送請求
Web Layer — Controller
驗證 Request DTO → 轉換為 Domain 物件
呼叫 Service
Application Layer — Service
使用 Domain 物件處理邏輯 → 透過 Port 介面存取資料
呼叫 Port 實作
Repository Layer — Repo
操作 DB / Cache / 外部服務
回傳結果
HTTP Response
Controller 組裝 Response DTO 回傳
